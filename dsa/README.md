# General Coding Track

This is the general-coding half of ML interviews: solve under time pressure, communicate the invariant, and test edge cases.

## Order

1. Arrays / hashing / two pointers
2. Linked lists / stacks / queues
3. Trees / graphs
4. Backtracking
5. Heaps / intervals
6. Dynamic programming

## Per problem protocol

- Restate inputs, outputs, and constraints.
- Give a brute-force baseline.
- State the invariant behind the optimized approach.
- Code in small pieces and test normal, empty, duplicate, and boundary cases.
- State time and space complexity.

Use `dsa/drills.md` as your problem queue. Re-implement the reference files from memory, then solve the linked variants.
